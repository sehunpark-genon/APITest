## 2. API별 상세 정리

## 2-1. 법제처 국가법령정보 Open API(현행법령 본문) - 자동승인

### 개요

법제처 국가법령정보 Open API는 국가법령정보센터에서 제공하는 법령 정보를 외부 시스템에서 조회하고 활용할 수 있도록 제공하는 공공 API이다.

국가법령정보 공동활용 서비스에서는 현행법령 정보를 조회할 수 있는 API를 제공하며, **법령 목록 조회**와 **법령 본문 조회** 방식으로 구분하여 활용할 수 있다.

현행법령 조회는 기준일에 따라 **공포일 기준과 시행일 기준으로 구분**된다. 공포일은 법령이 제정 또는 개정된 후 공식적으로 공표된 날짜를 의미하며, 시행일은 해당 법령이 실제 효력을 가지고 적용되기 시작하는 날짜를 의미한다. 따라서 실제 서비스에서 현재 적용되는 법령 정보를 기준으로 활용하려는 경우에는 시행일 기준 조회를 우선적으로 검토할 수 있다.

### 조회 방식

국가법령정보 Open API는 크게 목록 조회와 본문 조회 흐름으로 활용한다.

 1. 목록 조회

목록 조회는 조건에 맞는 법령 목록을 검색하는 API이다.

법령명, 검색어, 공포일자, 시행일자 등의 조건을 이용해 법령을 검색할 수 있으며, 조회 결과를 통해 법령명, 법령ID, 법령일련번호, 공포일자, 시행일자, 소관부처 등의 기본 정보를 확인할 수 있다.

목록 조회는 이후 본문 조회를 수행하기 위한 법령 식별값을 확보하는 단계로 활용된다.

 2. 본문 조회

본문 조회는 목록 조회를 통해 확보한 법령ID 또는 법령일련번호를 기준으로 특정 법령의 상세 내용을 조회하는 API이다.

본문 조회 결과에는 법령의 기본정보와 함께 조문, 항, 호, 목, 부칙, 별표 및 서식 정보가 포함될 수 있다.

별표 또는 서식이 별도 파일 형태로 제공되는 경우에는 응답 데이터 내에 HWP, PDF, 이미지 파일명 또는 다운로드 링크가 포함될 수 있다. 이 경우 해당 링크를 통해 별표·서식 파일을 별도로 수집할 수 있다.

### 공포일 기준 조회와 시행일 기준 조회

국가법령정보 Open API의 현행법령 조회는 공포일 기준과 시행일 기준으로 나누어 볼 수 있다.

| 구분 | 의미 | 활용 기준 |
| --- | --- | --- |
| 공포일 기준 | 법령이 공식적으로 공표된 날짜를 기준으로 조회 | 법령의 제정·개정 이력 및 공포 당시 정보를 확인할 때 활용 |
| 시행일 기준 | 법령이 실제 효력을 가지고 적용되는 날짜를 기준으로 조회 | 현재 적용 가능한 법령 본문을 확인할 때 활용 |

공포일과 시행일은 항상 동일하지 않을 수 있다. 예를 들어 특정 법령이 2026년 3월 1일에 공포되었더라도, 실제 시행일은 2026년 9월 1일로 정해질 수 있다. 이 경우 법령은 3월 1일에 공식 발표되었지만, 실제 적용은 9월 1일부터 이루어진다.

### 제공 형식

국가법령정보 Open API는 조회 결과를 HTML, XML, JSON 형식으로 제공한다.

## 2-2. 복지로 통합 API - 자동승인

### 개요

복지로 관련 데이터는 공공데이터포털을 통해 복지서비스 정보를 조회하거나 다운로드하여 활용할 수 있는 공공데이터이다.

본 검토에서는 복지로 관련 데이터를 **중앙부처복지서비스 API**, **지자체복지서비스 API**, **민간복지서비스정보 파일데이터**로 구분하여 정리한다.

**중앙부처복지서비스 API**와 **지자체복지서비스 API**는 Open API 방식으로 제공되며, 각각 복지서비스 목록조회와 상세조회 기능을 제공한다. **민간복지서비스정보**는 실시간 호출이 가능한 Open API 방식이 아니라 파일데이터 형태로 제공되므로, 파일을 다운로드한 뒤 자체 시스템에 적재하여 활용하는 방식으로 검토한다.

### 조회 방식

복지로 데이터는 제공 주체와 제공 방식에 따라 중앙부처복지서비스, 지자체복지서비스, 민간복지서비스정보로 나누어 수집할 수 있다.

1. 중앙부처복지서비스 API

중앙부처복지서비스 API는 공공데이터포털의 `한국사회보장정보원_중앙부처복지서비스` API를 활용하는 방식이다.

해당 API는 정부 각 중앙부처에서 전국민을 대상으로 제공하는 사회보장 급여 및 복지서비스 정보를 제공한다.

중앙부처복지서비스 API는 복지서비스 목록조회와 복지서비스 상세조회 기능으로 구분된다. 목록조회는 제공 중인 중앙부처 복지서비스 목록을 확인하는 데 사용하며, 상세조회는 특정 복지서비스의 지원대상, 선정기준, 신청방법 등 세부 내용을 확인하는 데 사용한다.

일반적인 활용 흐름은 다음과 같다.

```
중앙부처 복지서비스 목록조회
→ 서비스 식별값 확보
→ 중앙부처 복지서비스 상세조회
→ 지원대상, 선정기준, 신청방법 등 세부 정보 수집
```

1. 지자체복지서비스 API

지자체복지서비스 API는 공공데이터포털의 `한국사회보장정보원_지자체복지서비스` API를 활용하는 방식이다.

해당 API는 각 지방자치단체에서 지역 주민을 대상으로 제공하는 복지서비스 정보를 제공한다.

지자체복지서비스 API 역시 복지서비스 목록조회와 복지서비스 상세조회 기능으로 구분된다. 목록조회는 지자체별 복지서비스 목록을 확인하는 데 사용하며, 상세조회는 특정 지자체 복지서비스의 지원대상, 선정기준, 신청방법 등 세부 내용을 확인하는 데 사용한다.

일반적인 활용 흐름은 다음과 같다.

```
지자체 복지서비스 목록조회
→ 서비스 식별값 확보
→ 지자체 복지서비스 상세조회
→ 지원대상, 선정기준, 신청방법 등 세부 정보 수집
```

1. 민간복지서비스정보 파일데이터

민간복지서비스정보는 공공데이터포털의 `한국사회보장정보원_민간복지서비스정보` 파일데이터를 활용하는 방식이다.

해당 데이터는 민간기관에서 제공하는 복지서비스 정보를 파일 형태로 제공한다. 실시간 호출이 가능한 Open API 방식이 아니라 CSV 또는 Excel 등 파일데이터 형태로 제공되므로, 파일을 직접 다운로드한 뒤 내부 DB 또는 수집 저장소에 적재하여 활용하는 방식이 적합하다.

민간복지서비스정보는 중앙부처 및 지자체 복지서비스 API와 함께 활용할 경우, 공공 복지서비스뿐만 아니라 민간에서 제공하는 복지서비스까지 함께 검토할 수 있는 보조 데이터로 활용할 수 있다.

### 제공 API 및 데이터

복지로 관련 데이터는 다음과 같이 구분할 수 있다.

| 구분 | 데이터/API명 | 제공 방식 | 활용 기준 |

| 2 | 한국사회보장정보원_지자체복지서비스 | Open API(XML) | 지자체 복지서비스 목록조회 및 상세조회 |

## 2-3. 서민금융진흥원 API - 자동승인

### 개요

서민금융진흥원 API는 공공데이터포털을 통해 서민금융 상품 정보를 조회하여 활용할 수 있는 공공 API이다.

본 검토에서는 서민금융진흥원에서 제공하는 여러 공공데이터 중, 서민대출상품 정보를 조회할 수 있는 **`서민금융진흥원_대출상품한눈에 정보 서비스`**를 기준으로 한다.

해당 API는 다양한 서민대출상품의 핵심 정보를 한눈에 비교하고 조회할 수 있도록 제공되는 API이다. 금융상품명, 대출한도, 금리구분, 대출용도, 총 대출기간, 취급기관 등 주요 항목별 정보를 포함하고 있으며, 정부기관, 정책금융기관, 지방자치단체 등 다양한 기관에서 제공하는 금융상품을 확인할 수 있다.

또한 자금용도, 지원대상 등 수요자가 원하는 기준에 따라 서민대출상품을 검색할 수 있어, 사용자의 필요에 맞는 상품 정보를 탐색하고 비교하는 데 활용할 수 있다.

본 검토에서는 금융분야 마이데이터 테스트베드에서 제공되는 개인 금융정보 조회 API가 아닌, 공공데이터포털에서 제공되는 일반 공공데이터 API를 기준으로 한다.

### 조회 방식

`서민금융진흥원_대출상품한눈에 정보 서비스`는 조건에 맞는 서민대출상품 정보를 조회하는 방식으로 활용한다.

해당 API는 별도의 목록 조회와 상세 조회로 구분되기보다는, 하나의 상품 검색 API를 통해 서민대출상품 정보를 조회하는 구조로 볼 수 있다.

조회 시 자금용도, 지원대상 등 검색 조건을 활용할 수 있으며, 조회 결과를 통해 금융상품명, 대출한도, 금리구분, 대출용도, 총 대출기간, 취급기관 등 상품 비교에 필요한 정보를 확인할 수 있다.

일반적인 활용 흐름은 다음과 같다.

```
검색 조건 설정
→ 서민대출상품 정보 조회
→ 금융상품명, 대출한도, 금리구분, 대출용도, 대출기간, 취급기관 등 주요 항목 확인
→ 조건에 맞는 상품 정보 수집
```

공공데이터포털에서 제공되는 `서민금융진흥원_대출상품한눈에 정보 서비스`는 REST 방식의 Open API이며, XML 형식으로 제공된다.

해당 API는 무료로 제공되며, 업데이트 주기는 연 1회이다. 심의유형은 개발단계와 운영단계 모두 자동승인으로 안내되어 있다.

## 2-4. 법률 판례 API - 자동승인

### 개요

법률 판례 API는 국가법령정보 공동활용 서비스에서 제공하는 판례 정보를 외부 시스템에서 조회하고 활용할 수 있도록 제공하는 공공 API이다.

국가법령정보 공동활용 서비스에서는 판례 정보를 조회할 수 있는 API를 제공하며, **판례 목록 조회**와 **판례 본문 조회** 방식으로 구분하여 활용할 수 있다.

판례 목록 조회를 통해 사건명, 사건번호, 선고일자, 법원명 등의 기본 정보를 확인할 수 있으며, 판례 본문 조회를 통해 특정 판례의 판시사항, 판결요지, 참조조문, 판례내용 등을 확인할 수 있다.

### 조회 방식

법률 판례 API는 크게 목록 조회와 본문 조회 흐름으로 활용한다.

1. 목록 조회

목록 조회는 조건에 맞는 판례 목록을 검색하는 API이다.

사건명, 검색어, 법원종류, 법원명, 참조법령명 등의 조건을 이용해 판례를 검색할 수 있으며, 조회 결과를 통해 판례일련번호, 사건명, 사건번호, 선고일자, 법원명 등의 기본 정보를 확인할 수 있다.

목록 조회는 이후 본문 조회를 수행하기 위한 판례 식별값을 확보하는 단계로 활용된다.

1. 본문 조회

본문 조회는 목록 조회를 통해 확보한 판례일련번호를 기준으로 특정 판례의 상세 내용을 조회하는 API이다.

본문 조회 결과에는 판례의 기본정보와 함께 사건명, 사건번호, 선고일자, 법원명, 판시사항, 판결요지, 참조조문, 판례내용 등이 포함될 수 있다.

따라서 판례 데이터는 목록 조회를 통해 판례 식별값을 확보한 뒤, 본문 조회를 통해 실제 판례 전문과 주요 판단 내용을 수집하는 방식으로 활용할 수 있다.

### 목록 조회와 본문 조회

국가법령정보 공동활용의 판례 API는 목록 조회와 본문 조회로 나누어 볼 수 있다.

| 구분 | 의미 | 활용 기준 |
| --- | --- | --- |
| 목록 조회 | 조건에 맞는 판례 목록을 검색하는 조회 방식 | 판례 검색 및 본문 조회를 위한 판례일련번호 확보 |
| 본문 조회 | 특정 판례의 상세 내용을 조회하는 방식 | 판시사항, 판결요지, 참조조문, 판례내용 등 상세 정보 확인 |

판례 목록 조회에서는 검색범위를 지정할 수 있으며, 기본적으로 판례명 검색을 사용할 수 있고 필요에 따라 본문검색을 활용할 수 있다. 또한 법원종류, 법원명, 참조법령명 등의 조건을 이용해 검색 범위를 좁힐 수 있다.

### 제공 형식

법률 판례 API는 조회 결과를 HTML, XML, JSON 형식으로 제공한다.
