<외부망, DMZ, 망연계 구간, 내부망 흐름>

 외부망(연계 대상 API) : 연계 대상 외부 API는 법제처 국가법령정보(현행법령 본문), 복지로 통합 API, 서민금융진흥원 API, 대법원 판례 API 네 가지로 한정하고 모두 공개 데이터로서 개인신용정보·민감정보는 비포함된 상태임. 수집 대상은 반드시 이 공개 데이터 범위 안에서만 운영

 DMZ(외부 API 호출의 완결 지점) : 외부 API의 요청·응답 통신 세션은 반드시 DMZ 구간 안에서 완결되어야 하고, 내부망이 외부 API를 직접 호출하는 구성은 허용되지 않음. 수집된 데이터는 DMZ에서 정제·변경분 감지(Diff)를 수행하여 변경된 항목만 추출하고, JSON/_xml_/HWP/PDF 등 다양한 형식을 일관된 텍스트+메타데이터로 정규화. 특히 HWP·PDF 원본은 DMZ 단계에서 텍스트를 미리 추출·변환하여 JSONL 형식으로 스테이징한 후 망연계 구간으로 인계

 망연계 구간(기존 MCCUBE 재활용) : 망연계는 기구축된 MCCUBE 솔루션을 그대로 재활용하며 신규 장비를 도입하지 않고 데이터는 TCP 상시 세션이 아닌 파일 단위 단방향 전송 방식으로만 DMZ에서 내부망으로 이동. 전송 과정에서 악성코드 검사, CDR 무해화, 포맷·확장자 화이트리스트 검증, 해시 무결성 비교, 전송 로그 기록 등 기존 보안 통제 준수

 내부망(수신·임베딩·Vector DB 적재) : 내부 수신 에이전트가 수신 디렉토리의 파일을 감지하여 내부 Kafka 메시지 큐에 적재, 이 큐는 외부와 통신하지 않고 내부 처리 버퍼 역할만 수행. 임베딩은 폐쇄망 내부에 구축된 온프레미스 한국어 임베딩 모델만 사용하고, 외부 임베딩 서비스는 일절 호출하지 않고 생성된 벡터는 변경분만 Upsert 방식으로 Vector DB에 적재. 최종적으로 RAG 검색 및 온프레미스 LLM과 연결되어 최신 외부 데이터 기반의 생성형 AI 서비스를 제공
